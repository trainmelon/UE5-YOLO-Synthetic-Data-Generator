# UE5-YOLO-Synthetic-Data-Generator (虚拟环境自动化数据生成与标注系统)

## 项目简介
本项目旨在解决深度学习目标检测任务中，人工标注数据成本高、效率低的问题。通过 **Unreal Engine 5 (UE5)** 结合 **Cesium** 搭建高逼真度地理环境，利用 **Python** 脚本自动化控制相机视角、计算投影参数，从而批量生成带高精度标注框的合成数据集，并成功应用于 **YOLO** 模型的训练。

## 核心亮点
* **自动化生成：** 运用 Python 脚本替代人工手动框选，实现数据集的规模化生产。
* **高逼真度模拟：** 结合 Cesium 导入真实 3DTiles 底图与手工模型，缩小 Sim2Real（仿真到现实）差距。
* **端到端闭环：** 涵盖从“虚拟环境搭建 -> 自动化数据采集 -> YOLO 模型微调”的完整流程。

## 声明：关于 3D 资产与地理底图
遵循代码仓库的最佳实践，本仓库仅开源核心算法脚本、训练配置与模型结构。受限于 GitHub 容量限制及资产版权，**本仓库不包含以下大型二进制资产文件**：
* 真实地理环境的 3DTiles 底图数据。
* 场景中使用的精细 3D 模型（如：风机、轮船、电力杆塔、汽车等）。
*开发者若需复现完整渲染环境，请自行在 UE5 中导入相应的 Cesium 数据与相关静态网格体 (Static Meshes)。*

## 项目文件架构说明
* `data_create_MVP.py`：最小可行性测试脚本，用于验证 30 张图片生成逻辑与投影参数。
* `check_lables.py`：人工校验脚本，二次确认投影参数方程的准确性。
* `create_car.py`：模型生成脚本，根据蓝图样板线自动生成系列车辆模型。
* `data_generator.py`：核心生成脚本，基于验证逻辑自动化生成 5000 张规模的数据集。
* `data_clean.py`：数据清洗脚本，自动过滤无效框与空数据，并结合人工初步筛查。
* `data_divide.py`：数据集划分工具，按 8:1:1 比例自动切分训练集、验证集与测试集。
* `train.py`：YOLO 模型训练启动脚本。
* `test.py`：跨域验证脚本，对比分析 UE5 模拟图像与实景图像的推理差异。
* `v1_baseline/`：包含最佳匹配模型文件与训练过程的可视化评估照片。
* `Real_World_Test/` & `Real_World_Results/`：包含实景图片测试集及对应的推理结果对比图。

## 工程复盘与 Sim2Real 挑战 (Limitations)
本项目验证了基于 UE5 与 Cesium 构建合成数据集以训练 YOLO 模型的可行性，但在跨越“领域鸿沟（Domain Gap）”进行实景推理时，仍存在以下局限性 ：
1. **数据采样机制导致的先验过拟合：** 单目标追踪的视场捕捉逻辑会导致背景同类目标“隐性漏标”；此外，相机拍摄参数的扰动范围较窄，导致模型对目标的尺度与位置产生了先验过拟合，在面对真实多变航拍图像时泛化受限 。
2. **微小目标特征语义衰减：** 远景处的风机或轮船等微小目标，在 YOLOv8 骨干网络多次下采样后，其深层语义特征急剧衰减，导致远景地物置信度普遍偏低 。
3. **密集场景 NMS 抑制冲突：** 真实世界高密度场景下，由于合成数据集缺乏极度拥挤的重叠样本，后处理阶段的非极大值抑制（NMS）算法容易将正确预测框误判为冗余并强行剔除，造成严重漏检。
4. **算力与资产瓶颈导致的特征特化：** 受限于单机算力（RTX 3070 Ti 8GB），无法开启极限物理渲染参数；且选用的 3D 资产特征单一化，导致模型对特定材质和光影反馈产生了一定程度的过拟合 。

## 未来展望 (Future Work)
针对上述局限性，未来的优化方向包括：
* **全局实例分割与自动标注：** 弃用单点射线检测，改用 UE5 自定义深度通道渲染全局语义分割遮罩，实现 100% 像素级无漏标自动包围盒生成 。
* **域随机化增强 (Domain Randomization)：** 引入更激进的焦距变化、相机畸变与真实天气系统干扰，抹除模型的空间分布先验 。
* **Sim2Real 域适应微调：** 收集少量真实世界航拍数据进行迁移学习，弥合虚拟渲染纹理与真实物理环境之间的特征鸿沟。
* **实景三维建模引入：** 引入实景三维重建技术（如 NeRF 或 3D Gaussian Splatting）逆向生成高保真资产并导入 UE5，配合 PBR 物理级渲染，从根源消除纹理鸿沟
