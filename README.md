# UE5-YOLO-Synthetic-Data-Generator (虚拟环境自动化数据生成与标注系统)

## 项目简介
本项目旨在解决深度学习目标检测任务中，人工标注数据成本高、效率低的问题。通过 **Unreal Engine 5 (UE5)** 结合 **Cesium** 搭建高逼真度地理环境，利用 **Python** 脚本自动化控制相机视角、计算投影参数，从而批量生成带高精度标注框的合成数据集，并成功应用于 **YOLO** 模型的训练。

## 核心亮点
* **自动化生成：** 运用 Python 脚本替代人工手动框选，实现数据集的规模化生产。
* **高逼真度模拟：** 结合 Cesium 导入真实 3DTiles 底图与手工模型，缩小 Sim2Real（仿真到现实）差距。
* **端到端闭环：** 涵盖从“虚拟环境搭建 -> 自动化数据采集 -> YOLO 模型微调”的完整流程。

## 声明：关于 3D 资产与地理底图
遵循代码仓库的最佳实践，本仓库仅开源核心算法脚本、训练配置与模型结构。受限于 GitHub 容量限制及资产版权，**本仓库不包含以下大型二进制资产文件**：
* 真实地理环境的 3DTiles 底图数据。
* 场景中使用的精细 3D 模型（如：风机、轮船、电力杆塔、汽车等）。
*开发者若需复现完整渲染环境，请自行在 UE5 中导入相应的 Cesium 数据与相关静态网格体 (Static Meshes)。*

## 项目文件架构说明
* `data_create_MVP.py`：最小可行性测试脚本，用于验证 30 张图片生成逻辑与投影参数。
* `check_lables.py`：人工校验脚本，二次确认投影参数方程的准确性。
* `create_car.py`：模型生成脚本，根据蓝图样板线自动生成系列车辆模型。
* `data_generator.py`：核心生成脚本，基于验证逻辑自动化生成 5000 张规模的数据集。
* `data_clean.py`：数据清洗脚本，自动过滤无效框与空数据，并结合人工初步筛查。
* `data_divide.py`：数据集划分工具，按 8:1:1 比例自动切分训练集、验证集与测试集。
* `train.py`：YOLO 模型训练启动脚本。
* `test.py`：跨域验证脚本，对比分析 UE5 模拟图像与实景图像的推理差异。
* `v_baseline/`：包含最佳匹配模型文件与训练过程的可视化评估照片。
*  `Results/`：包含实景图片测试推理结果。

## 🚀 V2 核心架构升级 (Major Updates in V2)
在最新的 V2 版本中，我们针对 V1 基础模型在跨越“领域鸿沟（Domain Gap）”时暴露出的缺陷进行了底层逻辑重构，实现了工业级的合成数据量产：
1. **全屏视锥体穷举扫描 (Exhaustive Frustum Scanning)：** 彻底弃用单目标追踪的视场捕捉逻辑，引入全局实例遍历。对处于相机视锥体内且未被完全遮挡的所有目标进行 100% 穷举标注，从根源上消除了高密度场景（如停车场、风电场）下的“隐性漏标”问题。
2. **类别自适应动态焦距 (Class-Adaptive Dynamic Focal Length)：** 打破了固定的空间先验分布。针对不同体积特征的地物（如百米高耸的风机与扁平的汽车），代码会自适应匹配 1.5 倍至 20 倍的剧烈焦距拉伸，极大地丰富了模型对多尺度微小目标（Small Object Detection）的特征提取鲁棒性。
3. **高级脏数据清洗与面积豁免权 (Industrial-Grade Data Cleaning)：** 引入边缘截断判定机制，对距离画面边缘 0.5% 以内且占全屏面积低于 2% 的残缺特征进行自动化剔除，确保喂给 YOLO 模型的每一组坐标均为极高质量的纯血数据。V2 模型最终在极端视角下依然达到了 **mAP@0.5 = 83.9%** 的优异收敛成绩。

## 工程复盘与现存挑战 (Limitations)
尽管 V2 架构大幅提升了泛化能力，但在极致的 Sim2Real 实景推理中，仍存在以下物理与算法层面的局限：
1. **密集场景 NMS 抑制冲突：** 真实世界高密度场景下，由于合成数据集仍难以完美复刻极度拥挤的重叠样本，后处理阶段的非极大值抑制（NMS）算法容易将正确预测框误判为冗余并强行剔除，造成局部漏检。
2. **算力与资产瓶颈导致的特征特化：** 受限于单机显卡算力，无法无限制开启极限物理级渲染参数（如极致全局光照反射）；且选用的 3D 资产特征较为单一，导致模型对特定材质和光影反馈仍存在一定程度的过拟合。

## 未来展望 (Future Work)
针对上述局限性，本课题未来的优化与拓展方向包括：
* **实景三维建模引入 (Photogrammetry & NeRF)：** 引入实景三维重建技术（如 3D Gaussian Splatting）逆向生成高保真资产并导入 UE5，配合 PBR 物理基渲染，从根源消除虚拟纹理与物理现实的视觉鸿沟。
* **域随机化与实例分割 (Domain Randomization & Instance Segmentation)：** 引入更激进的相机畸变与真实天气系统干扰。同时探索 UE5 自定义深度通道，渲染全局语义分割遮罩，实现像素级的自动化标注。
* **Sim2Real 域适应微调 (Domain Adaptation)：** 收集少量真实世界航拍数据进行迁移学习，弥合特征鸿沟。
* **域随机化增强 (Domain Randomization)：** 引入更激进的焦距变化、相机畸变与真实天气系统干扰，抹除模型的空间分布先验 。

## ⚠️ 数据合规与开源声明 (Data Compliance & Open Source Declaration)
遵循行业开源规范与数据隐私安全准则，本仓库仅开源核心算法脚本、流水线架构及非敏感测试样例：
1. **模型与底图：** 受限于文件大小及安全性考量，3dtile底图不予以提供，静态网格体模型均来自与网络，并作简易的处理。
